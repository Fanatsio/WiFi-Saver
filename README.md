# WiFi Saver v1.1

**Автоматический мониторинг и перезапуск Wi-Fi при потере интернета на Windows**

Программа постоянно проверяет доступность интернета (пинг до 8.8.8.8:53) и при нескольких неудачных попытках автоматически перезапускает указанный Wi-Fi интерфейс через `netsh`.

Отличное решение для нестабильного Wi-Fi роутера, когда соединение «падает», но само не восстанавливается.

## ✨ Возможности

- Цветной вывод в консоль (зелёный — онлайн, жёлтый — оффлайн, красный — перезапуск)
- Проверка подключения через прямое TCP-соединение (надёжнее пинга)
- Автоматический перезапуск Wi-Fi адаптера
- Настраиваемое количество неудачных попыток перед перезапуском
- Работает только от имени **администратора** (требуется для `netsh`)
- Поддержка русского языка в именах интерфейсов

## 🚀 Как использовать

### 1. Скачивание и сборка

```bash
git clone https://github.com/твой-ник/wifi-monitor.git
cd wifi-monitor
```

Открой проект в **Visual Studio** (рекомендуется) и собери в конфигурации **Release x64** (или x86).

### 2. Настройка

Открой файл `main.cpp` и измени константу под своё подключение:

```cpp
const wchar_t* INTERFACE_NAME = L"Беспроводная сеть";   // ← Измени здесь!
```

Как узнать точное имя интерфейса:
- Нажми `Win + X` → **Диспетчер устройств** → **Сетевые адаптеры**
- Или выполни в CMD от администратора:
  ```cmd
  netsh interface show interface
  ```

### 3. Запуск

**Обязательно** запускай программу **от имени администратора**!

- Правой кнопкой по `WiFiMonitor.exe` → «Запуск от имени администратора»
- Или создай ярлык и в свойствах поставь галочку «Запускать от имени администратора»

## ⚙️ Настраиваемые параметры (в начале кода)

| Параметр                  | Значение по умолчанию | Описание |
|--------------------------|-----------------------|--------|
| `INTERFACE_NAME`         | "Беспроводная сеть"   | Имя Wi-Fi интерфейса |
| `CHECK_IP`               | "8.8.8.8"             | IP для проверки интернета |
| `CHECK_PORT`             | 53                    | Порт (DNS) |
| `CHECK_TIMEOUT`          | 3000                  | Таймаут проверки (мс) |
| `RETRY_BEFORE_RESTART`   | 2                     | Сколько раз подряд нет интернета → перезапуск |
| `SLEEP_INTERVAL`         | 5                     | Интервал проверки (секунды) |
| `POST_RESTART_WAIT`      | 10                    | Ожидание после перезапуска (секунды) |

## 📋 Требования

- Windows 10 / 11 (протестировано)
- Visual Studio 2019/2022 (или MinGW с поддержкой Winsock)
- Запуск **от имени администратора**

## 🛠 Сборка через командную строку (опционально)

```cmd
g++ -std=c++17 -O2 -Wall -o WiFiSaver.exe main.cpp -static -lws2_32 -static-libgcc -static-libstdc++
```

## Известные особенности

- Программа использует `netsh interface set interface "Имя" disable/enable` — это самый надёжный способ перезапустить Wi-Fi адаптер на Windows.
- После перезапуска ждёт 10 секунд, чтобы адаптер полностью поднялся.
- Работает в бесконечном цикле (можно свернуть в трей в будущих версиях).
